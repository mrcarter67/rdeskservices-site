Standards We Work Within

Every client engagement is scoped and delivered with applicable compliance frameworks in mind — from federal CUI protection to state accessibility requirements.

📋
NIST SP 800-171
CUI Protection
Aligned

NIST Special Publication 800-171 defines requirements for protecting Controlled Unclassified Information (CUI) in non-federal systems. RDesk aligns all managed environments with these 110 controls across 14 control families — making us eligible to support DoD contractors and agencies handling CUI.

  • Access control — least privilege, MFA, role-based access enforced in Entra ID
  • Audit and accountability — Splunk SIEM log collection, retention, and review
  • Configuration management — baseline configurations documented and enforced
  • Incident response — documented IR playbooks and post-incident reporting
  • System and communications protection — VLAN segmentation and encrypted transit
  • Risk assessment — periodic vulnerability scanning and remediation tracking
🔐
CMMC Level 1 & 2
DoD Cyber Maturity
Ready

The Cybersecurity Maturity Model Certification (CMMC) is the DoD's framework for assessing contractor cyber posture. Level 1 covers 17 basic safeguarding requirements (FAR 52.204-21); Level 2 maps to all 110 NIST 800-171 controls. RDesk is prepared to support clients at both levels and can assist with evidence collection for third-party assessments.

  • Level 1: All 17 FAR 52.204-21 practices implemented in managed environments
  • Level 2: 110 NIST 800-171 practices — documented, implemented, and reviewed
  • System Security Plan (SSP) documentation support for client environments
  • Plan of Action & Milestones (POA&M) tracking and remediation management
  • Evidence collection and audit support for C3PAO assessments
🛡️
CIS Controls v8
Implementation Group 2
Implemented

The CIS Critical Security Controls provide a prioritized set of actions for cyber defense. RDesk operates at Implementation Group 2 (IG2), covering all 18 control families across endpoint security, network monitoring, data protection, access management, and incident response.

  • Control 1 & 2: Asset inventory management for hardware and software
  • Control 4: Secure configuration for enterprise assets via Intune and GPO
  • Control 6: Access control management — Entra ID, Conditional Access, PIM
  • Control 8: Audit log management in Splunk with SIEM alerting
  • Control 13: Network monitoring and defense via FortiGate and Defender
  • Control 17: Incident response management with documented playbooks

Section 508 / ADA
Accessibility
Compliant

Section 508 of the Rehabilitation Act requires all federal agencies and their contractors to make electronic and information technology accessible to people with disabilities. All digital deliverables from RDesk — documents, web properties, and tools — are tested against WCAG 2.1 AA and Section 508 standards.

  • WCAG 2.1 Level AA conformance on all client-facing web deliverables
  • Accessible PDF and document output using proper tagging and reading order
  • Color contrast ratios meeting 4.5:1 (normal text) and 3:1 (large text) minimums
  • Keyboard navigability and screen reader compatibility testing
  • Voluntary Product Accessibility Template (VPAT) available upon request
🏛️
FISMA Awareness
Federal IS Mgmt
Aware

The Federal Information Security Modernization Act (FISMA) requires federal agencies and their contractors to maintain information security programs. While FISMA assessment and authorization is an agency responsibility, RDesk understands FISMA requirements and factors them into scoping, documentation, and delivery for all federal engagements.

  • FISMA Low / Moderate / High impact level awareness in scoping discussions
  • Alignment with NIST RMF (Risk Management Framework) for federal clients
  • Security control documentation in formats suitable for ATO packages
  • Continuous monitoring practices aligned to FISMA ongoing assessment requirements
🌐
Microsoft Zero Trust
Identity-First Security
Deployed

RDesk deploys Zero Trust architecture aligned to Microsoft's Zero Trust model and CISA's Zero Trust Maturity Model. We implement identity verification, device compliance enforcement, least-privilege access, and continuous monitoring across all managed Microsoft 365 and Azure environments.

  • Entra ID (Azure AD) as the authoritative identity plane — MFA enforced for all users
  • Conditional Access policies — device compliance, location, risk signal-based access
  • Privileged Identity Management (PIM) for time-bound admin access
  • Microsoft Defender XDR — endpoint, identity, email, and cloud app protection
  • Network micro-segmentation aligned to Zero Trust network access principles
  • CISA Zero Trust Maturity Model — pillar assessment and roadmap documentation
